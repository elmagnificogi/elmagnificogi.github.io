# 需求评审（子模块）

> 父 Skill：[SKILL.md](../SKILL.md) | 阶段：S3

## 硬性规则

- **禁止** 0-1 撰写或改写需求正文
- **允许**：审查、标歧义、找遗漏、对照工程标冲突
- 套件未初始化 → 先 [create-kit.md](create-kit.md)

## 步骤

1. wolai MCP 读需求文档；扫描 workspace 模块/接口
2. 按维度评审：完整性、歧义、一致性、可行性、范围
3. 在 wolai 需求页写入「AI 评审意见」章节，并追加行动记录
4. 输出评审报告（chat 摘要）

## 输出格式

```markdown
## 需求评审报告
### 总体评估
[可进入技术文档 / 需人工纠偏后重审]
### 理解摘要（请人工确认）
-
### 问题清单
| 严重度 | 位置 | 问题 | 建议 |
| 🔴 阻塞 | | | |
| 🟡 建议 | | | |
### 工程冲突
-
### 开放问题
| 问题 | 建议选项 |
```

## 纠偏循环

人标记理解错误 → 人补 wolai → 重读 → 再评审，直到人确认。

## 完成后

→ S4 人工纠偏 → S5 技术文档起草（[solo](../solo-workflow.md) / [team](../team-workflow.md)）
